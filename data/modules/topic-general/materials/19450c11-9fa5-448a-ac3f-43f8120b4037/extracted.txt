What is a stack?
• A stack is a Last In, First Out (LIFO) 
data structure
• Anything added to the stack goes on 
the “top” of the stack
• Anything removed from the stack is 
taken from the “top” of the stack
• Things are removed in the reverse 
order from that in which they were 
inserted

Some uses of stacks
Stacks are used for:
– Any sort of nesting (such as tracking parentheses)
– Evaluating arithmetic expressions (and other sorts of expressions)
– Implementing function or method calls (built-in to virtually every programming language)
– Keeping track of choices yet to be made, or steps yet to be done.
– Keeping track of previous choices (as in search backtracking algorithms)

Implementing a Stack class using a built-in Python list
push method
Adds an item to the top of the stack.
Process:
First, self.__top is incremented by 1 to point to the next empty slot.
Then, item is placed at the position indicated by self.__top in self.__stacklist.
pop method
Removes and returns the item at the top of the stack.
Process:
top is set to the value at the self.__top index.
The self.__top pointer is then decremented by 1 to move down one level in the stack.
The method returns the item that was removed.

peek method

Critique of Stack
Popping an Empty Stack (Stack Underflow):
Trying to pop an item from an empty stack would cause an underflow.
if self.__top < 0:
    raise Exception("Stack underflow")
The updated code snippet checks if self.__top < 0 in the pop() method. If self.__top is less than 0, it means the 
stack is empty.
Raising an Exception with a message like "Stack underflow" prevents the method from attempting to access an 
invalid index, avoiding a runtime error.

Critique of Stack
Pushing to a Full Stack (Stack Overflow):
Trying to push an item to a Full stack would cause an overflow.
if self.__top == len(self.__stackList) - 1:
    raise Exception("Stack overflow")
The updated push() method checks if self.__top is equal to len(self.__stackList) - 1, which would indicate 
the stack is already at maximum capacity.
If this condition is true, an Exception with a message like "Stack overflow" is raised. This prevents pushing 
more items than the stack can hold and avoids an IndexError.

Critique of Stack
Interfering with Garbage Collection (gc):
self.__stackList[self.__top] = None
When an item is popped, it’s essential to set self.__stackList[self.__top] to None. 
This clears the reference to the removed item, allowing Python's garbage collector (gc) to free the memory 
associated with the removed object, especially for complex data types or objects.
By setting the popped position to None, you avoid potential memory leaks, which could happen if references 
are retained.

Application: Using a stack to make sure that 
brackets and parens are balanced
• ( [ ] ( { ( ) } [ ( ) ] ) ) is balanced;
( [ ] ( { ( ) } [ ( ) ) ] ) is not
• Simple counting is not enough to check the balance
• You can do it with a stack: going left to right,
– If you see a (, [, or {, push it on the stack
– If you see a ), ], or }, pop the stack and check whether you popped the corresponding (, [, or {
– When you reach the end, check that the stack is empty

Application: Expression evaluation
• Almost all higher-level programming languages let you evaluate complex expressions, such 
as 3*x+y+4*z
• You may occasionally need to evaluate expressions in your own program, without benefit 
of a compiler
• Our typical notation where operands appear on the left and right sides of an operator, is 
known as “infix notation”
• In contrast,  “postfix notation” places the two operands to the left of the operator. 
• For example: 
• Infix notation:                3 * x + y + 4 * z
• Postfix (RPN) notation: 3 x * y + 4 z * +

Performing calculations
• To evaluate any expression, such as 1+2*3+4 we make two 
passes through the expression:
• Convert the “infix notation” expression to “postfix notation” (also 
known as “Reverse Polish Notation”, RPN)
• Evaluate the postfix expression to compute its value
• Each pass uses a stack
